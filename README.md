# Анализ дефектов стальных плит

## Выбор датасета
Для проекта выбран датасет **[Steel Plates Faults](https://www.kaggle.com/datasets/eouedraogo4/steel-plates-faults)** с Kaggle.  
Датасет решает задачу многоклассовой классификации дефектов стальных плит и идеально подходит для проверки гипотез.

## Гипотезы (сформулированы до начала экспериментов)
1. F1-macro будет как минимум на 0.1 ниже accuracy, потому что классы сильно несбалансированы.
2. Использование class_weight='balanced' в Random Forest повысит F1-macro.
3. Удаление признаков с корреляцией > 0.95 не ухудшит качество модели.
4. Random Forest покажет лучшее F1-macro, чем Logistic Regression, из-за нелинейных зависимостей.
5. Отбор 10 лучших признаков через SelectKBest(f_classif) сохранит не ниже 95% от исходного F1-macro.

## Результаты проверки гипотез
1. Не подтвердилась. У LR accuracy = 0.704, F1-macro = 0.736; у RF accuracy = 0.761, F1-macro = 0.782. F1-macro оказался выше accuracy, что говорит о том, что модели достаточно хорошо справляются даже с редкими классами.
2. Подтвердилась. F1-macro вырос с 0.782 (baseline) до 0.787, то есть подтверждает пользу балансировки при дисбалансе классов.
3. Подтвердилась. После удаления 5 скоррелированных признаков F1-macro немного увеличился до 0.790, что говорит об уменьшении избыточности без потери качества.
4. Подтвердилась. Random Forest(f1 = 0.782) превзошёл Logistic Regression(f1 = 0.736), что работает с предположением о наличии нелинейных зависимостей в данных.
5. Не подтвердилась. F1-macro на 10 признаках составил всего 0.129, что значительно ниже порога 0.743 (95% от baseline). Это значит, что линейный метод отбора признаков(f_classif) не подходит для этой задачи.
